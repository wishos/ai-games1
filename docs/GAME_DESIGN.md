## 📐 数值平衡

### 玩家属性成长

| 等级 | HP | MP | ATK | DEF | SPD | LUK | 升级所需EXP |
|------|-----|-----|------|------|------|------|-----------|
| 1 | 120 | 30 | 14 | 6 | 5 | 3 | - |
| 2 | 132 | 33 | 16 | 7 | 6 | 4 | 60 |
| 3 | 144 | 36 | 18 | 8 | 7 | 5 | 141 |
| 4 | 156 | 39 | 20 | 9 | 8 | 6 | 250 |
| 5 | 168 | 42 | 22 | 10 | 9 | 7 | 383 |
| 10 | 240 | 57 | 34 | 16 | 14 | 12 | 1581 |
| 20 | 360 | 84 | 54 | 26 | 24 | 22 | 5944 |

公式：
- `HP增长` = `maxHp + 10` 每级
- `MP增长` = `maxMp + 3` 每级
- `ATK增长` = `atk + 2` 每级
- `DEF增长` = `def + 1` 每级
- `SPD增长` = `spd + 1` 每级
- `LUK增长` = `luk + 1` 每级
- `升级EXP` = `floor(level^1.5 * 60)`

### 战斗公式

- **普攻伤害** = `ATK - 敌人DEF + random(-3, 3)`，最小为1
- **暴击** = `random(100) < LUK * 2`，暴击伤害 x2
- **技能伤害** = `ATK * 1.8 - 敌人DEF + random(-5, 5)`
- **逃跑成功率** = 60%
- **防御减伤** = 50%

## 🌲 职业技能树

每个职业拥有独立技能树，分为**核心技能**（T1，初始解锁）、**进阶技能**（T2，Lv10解锁）、**终极技能**（T3，Lv25解锁）、**觉醒技能**（T4，Lv40解锁）四层。
每层可学习2-3个技能，需选择路线深入，最多同时激活T1+T2各1个+T3共1个。

### 装备强化系统

#### 强化机制

```
强化等级: +0 ~ +15
强化材料: 强化石 (各品质对应等级强化石)
强化费用: 金币 (随强化等级递增)
强化成功率: 100% until +10, then decreasing

| 强化等级 | 成功率 | 费用(金币) | 所需强化石 |
|---------|--------|-----------|-----------|
| +1~+3  | 100%  | 100/级    | 普通强化石 |
| +4~+6  | 100%  | 300/级    | 优秀强化石 |
| +7~+9  | 100%  | 600/级    | 稀有强化石 |
| +10    | 80%   | 1000      | 史诗强化石 |
| +11    | 65%   | 1500      | 史诗强化石 |
| +12    | 50%   | 2000      | 传说强化石 |
| +13    | 35%   | 3000      | 传说强化石 |
| +14    | 20%   | 5000      | 传说强化石 |
| +15    | 10%   | 10000     | 神器强化石 |
```

#### 强化保护机制

```
【保护符】
- 强化失败时，装备不降级，保留当前强化等级
- 强化+10以上必须使用，否则失败回+9
- 获得方式: 制作/商店购买/活动奖励

【祝福系统】
- 每次强化失败，积累"祝福值"+15%/+12%/+10%/+7%/+5%/+3% (对应+10~+15)
- 祝福值满时，下次强化必定成功
- 祝福值在换装备时清零
```

#### 强化属性加成

```
每强化一级，装备基础属性按以下比例提升:
+1~+5:  每级 +3%
+6~+10: 每级 +5%
+11~+15: 每级 +8%

示例: 一把紫色史诗剑基础ATK=250
+10时: 250 × (1.03^5 × 1.05^5) = 250 × 1.47 = ATK+367
+15时: 在+10基础上 × (1.08^5) = 367 × 1.47 = ATK+540
```

- 需要材料: 同名装备×1 + 进阶石×5 + 金币

### 敌人行为AI设计

```
AI决策优先级:
1. 血量危险 (<20%) → 使用治疗/逃跑技能

3. 目标有debuff可利用 → 执行收割
4. 普通攻击间隔 → 普攻

特殊机制:
- 群体怪物: 50%概率集火血量最低目标，50%概率随机目标
- 精英怪: 每3回合释放一次特殊技能
- Boss: 阶段转换时必有特殊动画和台词
```

### 战力计算公式

```
基础战力 = ATK + DEF + HP/10 + MP/5 + SPD*2 + LUK*3

装备战力 = Σ(装备ATK + 装备DEF*2 + 装备HP/20)

技能战力 = 已解锁技能数 * 20 + 技能等级总和 * 10

称号战力 = Σ(称号附加属性等效战力)

公会战力加成 = 公会等级 * 50

综合战力 = (基础战力 + 装备战力 + 技能战力 + 称号战力) * (1 + 公会加成/1000)
```

## 🌐 公会系统

### 公会概述

冒险者公会是玩家社交与成长的核心平台。每个玩家可加入一个公会，参与公会活动获取专属奖励。

```
公会等级: Lv1 - Lv10
- 每级需要公会经验: Lv * 5000
- 每周公会战结束后结算
- 满级公会可容纳50名成员
```

### 公会建筑

| 建筑 | 解锁等级 | 功能 | 升级效果 |

| 竞技场 | Lv6 | 公会战匹配入口 | 解锁3v3对战 |
| 图书馆 | Lv7 | 解锁成就积分兑换 | 兑换目录+1/级 |
| 龙穴入口 | Lv8 | 团队副本「龙之巢穴」挑战 | Boss等级+1/级 |

### 战力计算公式

```

装备战力 = Σ(装备ATK + 装备DEF*2 + 装备HP/20)

技能战力 = 已解锁技能数 * 20 + 技能等级总和 * 10

称号战力 = Σ(称号附加属性等效战力)

公会战力加成 = 公会等级 * 50

综合战力 = (基础战力 + 装备战力 + 技能战力 + 称号战力) * (1 + 公会加成/1000)
```

## 🎵 音乐与音效设计

### 音效风格定位

```
【整体风格】东方武侠 + 奇幻冒险 + 叙事性BGM
- 核心: 中国传统乐器（古琴/笛/箫/琵琶/二胡/锣鼓）
- 辅助: 西方奇幻管弦乐（竖琴/小提琴/大号/定音鼓）
- 融合: 电子音色点缀（用于魔法/科技元素）

【音乐层级】
Layer 1 (基础层): 持续性环境音（风声/流水/城市喧嚣）
Layer 2 (主旋律层): 主要音乐主题（由主乐器演奏）
Layer 3 (情绪层): 动态情绪变化（根据战斗/剧情走向）
Layer 4 (点缀层): 特殊场景音效（雷雨/鸟鸣/人群）
```

### 音乐制作规格

```
【音频格式】
- BGM: OGG Vorbis, 44.1kHz, 192kbps, 立体声
- 音效: OGG Vorbis, 44.1kHz, 128kbps, 单声道（定位音效）/立体声
- 环境音: WAV, 48kHz, 24bit

【BGM长度】
- 城镇BGM: 180-240秒（可无缝循环）
- 战斗BGM: 120-180秒（配合战斗节奏）
- Boss BGM: 240-360秒（三阶段无缝衔接）
- 剧情BGM: 按剧情长度（60-180秒）

【循环点设计】
- 循环过渡时间: 2-4秒淡入淡出
- 循环点位置: 选择情绪平稳段落（非高潮/低潮）
- 自动检测: 音乐系统自动寻找最佳循环点

【动态混音】
每个场景BGM预制4个混音版本：
- 安静版: 基础层(60%音量) + 点缀层(40%)
- 平衡版: 基础层(80%) + 情绪层(60%)
- 激烈版: 全三层(100%/80%/60%)
- 终极版: 全层+额外乐器(120%/100%/80%)

战斗系统根据战斗强度实时切换混音版本
```

## ⚖️ 游戏平衡性持续调整方案

### 数值平衡监控体系

```
【监控指标】
| 指标 | 理想范围 | 告警阈值 | 调整策略 |
|------|----------|----------|----------|
| 职业出场率 | 10-15%各职业 | <5%或>25% | 调整职业技能/数值 |
| Boss平均击杀时间 | 5-15回合 | <3或>25回合 | 调整Boss属性/AI |
| 道具使用频率 | 品类均衡 | 某类>60% | 调整效果/价格 |
| 刷装时间成本 | 1-3小时/件紫装 | >5小时 | 提升掉率 |
| 玩家等级分布 | 符合预期曲线 | 偏离>20% | 调整经验获取 |
| 流失率节点 | 主要在第3-5层 | 某层>40%流失 | 检查难度曲线 |

【数据采集】
- 战斗数据: 每场战斗结束后上报（匿名）
- 经济数据: 每周抽样5%玩家账户
- 行为数据: 定期(季度)玩家调研问卷
```

### 数值调整机制

#### 玩家属性调整
```
【常规调整周期】
- 每2周进行一次小范围数值审查
- 每季度进行一次大规模平衡更新

【调整原则】
1. 只调整数值，不改变技能机制（保持玩家预期）
2. 任何削弱调整需同步提供补偿
3. 调整幅度单次不超过±15%
4. 敏感调整(如nerf)需在测试服先行

【禁止事项】
- ❌ 不在付费内容上做削弱
- ❌ 不在赛季中期改变核心数值
- ❌ 不同时调整超过3个职业
```

#### 装备/掉落调整
```
【掉落率调整触发条件】
- 某装备/道具持有率<10%且获取难度>5小时: 提升掉率
- 某装备/道具持有率>60%且<2周内有溢出: 降低掉率或提升替代品价值

【装备数值调整】
- 单件装备ATK/DEF调整不超过±10%
- 套装属性调整需在版本更新中说明
- 任何装备调整有30天"保护期"（已获得玩家不受影响）
```

### 分级响应机制

```
【绿区 - 常规监控】
- 监控频率: 每日自动报表
- 响应方式: 记录、趋势分析
- 例: 某职业出场率12% → 正常，无需动作

【黄区 - 关注】
- 监控频率: 每3天人工审查
- 响应方式: 计划性调整，纳入下月更新
- 例: 某职业出场率降至7% → 计划加强

【橙区 - 预警】
- 监控频率: 每日人工审查
- 响应方式: 紧急热修复（若严重）/纳入下周更新
- 例: Boss通过率<20%，玩家社区负面情绪上升 → 热修复

【红区 - 紧急】
- 监控频率: 实时监控
- 响应方式: 24小时内热修复 + 补偿发放
- 例: 某技能导致无限刷钱Bug → 紧急修复 + 回档/补偿
```

### 玩家反馈响应流程

```
【反馈收集渠道】
- 游戏内反馈按钮 → 提交工单系统
- Discord/Reddit社区 → 每周汇总
- Steam评论 → 每月分析关键词
- 官方问卷 → 季度发放

【反馈处理流程】
1. 玩家提交反馈 → 自动分类（数值/体验/Bug/建议）
2. 24小时内: 客服回复确认收到
3. 7天内: 开发团队评估可行性
4. 14天内: 决定是否纳入开发计划
5. 纳入更新后: 在更新日志中致谢（需玩家同意）

【"策划回应"机制】
每月在官方社区发布"策划面对面"帖：
- 公布当前平衡数据
- 解释近期调整思路
- 预告下期调整方向
- 征集玩家意见
```

### 测试服（PTR）制度

```
【测试服开放时间】
- 新版本上线前14天开放
- 所有玩家可申请参与（名额限制10000人）

【测试服激励】
- 提交有效Bug报告: Steam库随机游戏
- 参与平衡调研: 限定称号「测试先锋」
- 发现重大Bug: Steam游戏优惠券

【测试数据使用】
- 收集数据仅用于平衡分析
- 不与正式服数据混合
- 测试服成绩不计入正式服成就
```

### 赛季制平衡策略

```
【赛季设定】
- 每个赛季持续3个月
- 赛季之间允许数值较大调整
- 赛季制内容: 排行榜重置 / 专属奖励 / 新增系统测试

【赛季间调整】
- 新赛季开始时公布完整数值变动
- 上赛季数据异常职业获得「适应buff」（+5%伤害，持续半个赛季）
- 重大机制调整（如技能重做）仅在赛季交接时进行

【赛季奖励赛季末补偿】
- 若某职业连续2个赛季表现垫底 → 下赛季发放补偿道具
- 若某装备连续2个赛季无人问津 → 下赛季提升基础属性15%
```

### 经济系统平衡

```
【通货膨胀监控】
- 每月统计金币产出/消耗比率
- 目标比率: 产出/消耗 = 1.0 ~ 1.1（轻微通胀可接受）

【通胀应对措施】
| 通胀等级 | 成因 | 应对 |
|----------|------|------|
| 轻度(<5%/季) | 正常消耗不足 | 新增高价商品/服务 |
| 中度(5-15%/季) | 刷金Bug/产出过高 | 修复+增加消耗点 |
| 重度(>15%/季) | 严重刷金 | 紧急修复+历史回档 |

【消耗点设计】
- 高级强化（+10以上）
- 稀有材料合成
- 服装/外观购买
- 坐骑/宠物系统
- 房屋装饰系统（未来扩展）
```
